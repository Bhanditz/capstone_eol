11     # These all have intelligent(ish) default values, so just specify those values that you feel are really salient. Note that a TC will
 
12     # NOT have a map or an IUCN status unless you specify options that create them.
 
13     #
 
14     #   +attribution+::
 
15     #     String to be used in scientific name as attribution
 
16     #   +bhl+::
 
17     #     Example: [{:publication => 'Foobar', :page => 23}, {:publication => 'Bazboozer', :page => 78}]
 
18     #   +canonical_form+::
 
19     #     String to use for canonical form (all names will reference this)
 
20     #   +comments+::
 
21     #     Array of hashes.  Each hash can have a +:body+ and +:user+ key.
 
22     #   +common_names+::
 
23     #     Array of strings to use for the common names.
 
24     #   +depth+::
 
25     #     Depth to apply to the attached hierarchy entry.  Don't supply this AND rank.
 
26     #   +event+::
 
27     #     HarvestEvent to associate the taxon to.
 
28     #   +flash+::
 
29     #     Array of flash videos, each member is a hash for the video options.  The keys you will want are
 
30     #     +:description+ and +:object_cache_url+.
 
31     #   +gbif_map_id+::
 
32     #     The ID to use for the Map Data Object.
 
33     #   +id+::
 
34     #     Forces the ID of the TaxonConcept to be what you specify, useful for exemplars.
 
35     #   +vetted+::
 
36     #     Modifies taxon concept vetted status. Can be 'trusted', 'untrusted', 'unknown', default is 'trusted'
 
37     #   +images+::
 
38     #     Array of hashes.  Each hash may have the following keys: +:description+, +:hierarchy_entry+,
 
39     #     +:object_cache_url+, +:taxon+, +:vetted+, +:visibility+ ...These are the args used to call #build_data_object
 
40     #     ALSO, the :images parameter may be set to :testing, which will create one image with lots of comments, as
 
41     #     well as one image of each vetted/visibility status, which is... uhhh... good for testing?
 
42     #     DEFAULT value is simply two images.
 
43     #   +italicized+::
 
44     #     String to use for preferred scientific name's italicized form.
 
45     #   +iucn_status+::
 
46     #     String to use for IUCN description, OR just set to true if you want a random IUCN status instead.
 
47     #   +biomedical_terms+::
 
48     #     Set to true (default is false) if you want the TOC to include an entry for Medical Concepts.  Note that
 
49     #     this is the "LigerCat Tag Cloud" entry.
 
50     #   +parent_hierarchy_entry_id+::
 
51     #     When building the associated HierarchyEntry, this id will be used for its parent.
 
52     #   +rank+::
 
53     #     String form of the Rank you want this TC to be.  Default 'species'.
 
54     #   +scientific_name+::
 
55     #     String to use for the preferred scientific name.  The first in the list will be "preferred"
 
56     #   +toc+::
 
57     #     An array of hashes.  Each hash may have a +:toc_item+ key and a +:description+ key. Note that there are
 
58     #     some toc entries that you CANNOT affect here, like common_names, bhl, or biomedical_terms, since they are
 
59     #     "special" and require more work.  See the options of those names for more information.
